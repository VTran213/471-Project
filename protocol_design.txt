PROTOCOL DESIGN: Simplified FTP Project

OVERVIEW:
This program uses two TCP connections: a control channel for commands and responses and a data channel for file and directory transfers. All messages use a 10 byte header followed by the data.

COMMANDS (Client to Server):

The client sends commands as TWO messages over the control channel:

1. Command Message:
- "ls"
- "get <filename>"
- "put <filename>"
- "quit" (is sent alone and doesn't require a port message)

2. Data Port Message:
- The ephemeral port number as a string (i.e. "12345")

RESPONSES (Server to Client):

All responses are framed messages:

SUCCESS <message>
FAILURE <message>

DATA CHANNEL:

- Client opens an ephemeral port and sends it to the server
- Server connects to that port
- Data is transferred
- Connection is closed after transfer

COMMAND FLOW:

LS:
Client sends "ls"
Client sends data port

Server sends SUCCESS
Server connects
Server sends directory listing
Server sends final SUCCESS

GET:
Client sends "get <filename>"
Client sends data port
Server sends SUCCESS or FAILURE

If SUCCESS:
Server connects
Server sends file
Server sends final SUCCESS

PUT:
Client sends "put <filename>"
Client sends data port

Server sends SUCCESS
Server connects to data port
Client sends file
Server sends final SUCCESS

QUIT:
Client sends "quit"
Server sends SUCCESS goodbye
Server closes connection
